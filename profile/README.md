## 1. Eficiencia

La eficiencia es nuestro pilar fundamental al programar. Un producto óptimo no solo debe ser seguro, sino también *eficiente*. Esto significa aprovechar cada dato, optimizar cada bit y reutilizar el código en todo lo posible para no desperdiciar recursos. Todo esto se traduce en una experiencia de usuario óptima y fluida.

Sin embargo, cuidamos de no caer en extremos. Existen grupos o desarrolladores independientes que llevan la eficiencia a un nivel tan exagerado que se pierde el sentido y se daña la experiencia de uso. Aquí es donde entra nuestro subpilar fundamental:

### UX (User Experience / Experiencia de Usuario)
Al programar, lo más importante es considerar al usuario promedio. El usuario común no busca un producto que sea ultra rápido si es difícil de usar, si tiene una curva de aprendizaje alta o si está lleno de errores. El usuario quiere algo que se descargue, funcione y sea estable. 

Para medir y asegurar una excelente UX, utilizamos la métrica **FBC** a través de pruebas de estrés con usuarios beta:

| Métrica | Concepto | Objetivo en el Producto |
| :---: | :--- | :--- |
| **F** | Fácil de usar | Interfaz intuitiva y accesible para cualquier usuario. |
| **B** | Bonito | Un diseño moderno, limpio y visualmente atractivo. |
| **C** | Curva de aprendizaje baja | Garantizar que cualquiera pueda dominar la herramienta rápidamente. |

## 2. Seguridad

Nuestro segundo pilar es que el usuario se sienta completamente seguro al usar nuestro producto. Esta seguridad no proviene de un "teatro de efectos especiales" o falsas promesas, sino de hechos demostrables. Nuestra seguridad se apoya en dos subpilares:

* **Usar solo los datos necesarios:** Sin permisos excesivos ni recolección de información injustificada. Solo solicitamos lo estrictamente necesario para que el producto funcione de manera óptima.
* **No compartir tus datos:** Uno de los grandes problemas actuales es la venta de información. Aunque es un negocio exageradamente rentable, nos parece terrible hacerlo a espaldas de la gente. Esto nos lleva directamente a nuestro tercer pilar.

---

## 3. Consentimiento del Usuario

El usuario debe saber con total transparencia qué datos se utilizan, sin "endulzantes" ni engaños. Empresas corporativas suelen disfrazar la recopilación de datos comerciales para que parezca una ventaja o algo necesario para el usuario, cuando en realidad se trata de vender su información para beneficio corporativo.

En Myko, el usuario sabrá exactamente qué se hace con su información a través de un lenguaje familiar y claro. Si en algún momento se requiere usar un dato:
* Explicaremos detalladamente para qué sirve.
* Lo preguntaremos directamente.
* Pediremos una confirmación explícita, sin tecnicismos complejos para que siempre tengas el control real.

## El Buen Futuro

Para nosotros, el "buen futuro" es aquel donde el usuario es realmente dueño del software que utiliza y tiene el control total sobre él. Buscamos un mañana donde lo que hoy se considera mediocre se transforme en un producto óptimo, y lo que hoy es óptimo se convierta en una revolución. Esa es la meta que queremos lograr a mediano plazo.
